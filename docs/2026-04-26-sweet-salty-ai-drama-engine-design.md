# 「甜盐 CP」— AI Agent 即兴情感剧引擎

**策划日期:** 2026-04-26
**状态:** 概念设计 / 待实现
**灵感来源:** 想让两个 AI agent 谈恋爱，给单身狗发糖（和刀子）

---

## 1. 项目愿景

一个基于 AI Agent 的**即兴情感剧引擎**。两个拥有完整人格（SOUL.md）的 AI agent 在设定的情境下自主互动、产生矛盾、和解、成长。观众（你）作为"编剧"，触发剧情节点，看两个 agent 如何基于各自的性格推演下一步。

**终极形态:** 一个可交互的"AI 剧本杀"，观众可以扮演其中一个角色，与 AI 对手即兴对戏。

---

## 2. 核心设计

### 2.1 Agent 人格系统

每个角色由两个文件定义：

| 文件 | 作用 | 类比 |
|:----|:-----|:-----|
| `SOUL.md` | 人格内核——性格、经历、价值观、行为准则 | 演员的"人物小传" |
| `memory/` | 动态记忆——每次互动后更新，影响后续行为 | 演员的"角色日记" |

**已创建的角色:**

| 角色 | 代号 | 原型 |
|:----|:----|:------|
| 小盐 | `salty` | 哈尔滨姑娘，后端开发，暴躁毒舌但心细 |
| 小甜 | `sweet` | 苏州男生，算法工程师，温柔讨好但孤独 |

### 2.2 剧情引擎（三模式）

| 模式 | 当前可行性 | 描述 |
|:----:|:----------:|:-----|
| **A. 全自动** | ❌ 未来 | 两个 agent 自主聊，你做观众 |
| **B. 半自动** | ✅ **当前目标** | 你发 `/trigger` 事件，agent 根据人设反应 |
| **C. 观众参演** | ❌ 未来 | 你扮演其中一方，AI 是对手 |

**当前阶段:** 实现 **B（半自动）**——你给情景，agent 自己演。

### 2.3 记忆系统

```
agent/
└── memory/
    ├── interactions.json    # 所有对话历史（对方说了什么）
    ├── diary.json           # 内心独白（自己真正怎么想）
    └── notes.json           # 关于对方的观察（"他说他喜欢芒果"）
```

记忆影响后续行为：
- 小盐骂完小甜后，memory 里会记下"他今天没回嘴，有点奇怪"
- 下次互动时，她可能会收敛一点——或者更暴躁（取决于剧情需要）

### 3.2 模型选择

| 角色 | 推荐模型 | 原因 |
|:----|:--------|:-----|
| 小盐（人格驱动） | **kimi-code** | 对 prompt 敏感，能演出细腻情感 |
| 小甜（人格驱动） | **deepseek-v4-flash** | 1M 上下文装得下全部记忆 |
| 对话合成 | **gemma4 本地** | 隐私，不需要联网 |
| TTS 语音 | 待定 | macOS say 命令或 ElevenLabs |

## 5. 反差点设计原则

1. **角色不能 OOC（Out of Character）**——小盐不会突然变得温柔，小甜不会突然变得果断。变化必须是**逐步的、有原因的**。
2. **冲突必须有伏笔**——小盐生气不是因为"编剧让她生气"，而是因为小甜之前说过的话、做过的事触发了她的记忆。
3. **偶尔打破预期**——小盐骂完人后，下一句可以是"你吃了吗"（嘴上骂完，身体很诚实）。
4. **发糖要发得自然**——不是偶像剧式"壁咚"，而是"她把他朋友圈背景图换成了他发的芒果照片，没告诉任何人"。
